# 技术选型与架构规范

> 版本：v1.0 | 更新日期：2026-06-08

## 技术选型

### 桌面框架：Electron

| 评估维度 | 说明 |
|----------|------|
| 剪贴板 API | 内置 `clipboard` 模块，支持文字/图片读写 |
| 系统托盘 | 内置 `Tray` 模块，原生支持 Windows 托盘 |
| 全局快捷键 | 内置 `globalShortcut` 模块 |
| UI 开发 | HTML/CSS/JS，卡片式 UI 灵活美观 |
| 打包分发 | `electron-builder` 一键生成 Windows 安装程序 |
| 生态系统 | npm 生态成熟，第三方包丰富 |
| 学习成本 | 前端技术栈即可开发，维护成本低 |

### 数据库：sql.js

| 评估维度 | 说明 |
|----------|------|
| 性能 | 纯 JavaScript SQLite 实现，无需 C++ 编译，适合几千条记录的桌面应用 |
| 部署 | 零依赖，跨平台兼容，无需 Visual Studio 编译工具 |
| 容量 | 单文件数据库，几千条记录无压力 |
| 可靠性 | 基于 Emscripten 编译的 SQLite，成熟稳定 |

### 图片处理：Electron nativeImage

| 评估维度 | 说明 |
|----------|------|
| 缩略图 | 使用 Electron 内置 `nativeImage.resize()` 生成卡片预览图 |
| 性能 | 避免额外图片处理依赖，对单张截图/图片处理足够快 |
| 格式支持 | 依赖 Electron 剪贴板图片读取，统一保存为 PNG |
| 部署 | 无第三方图片处理依赖，减少安装包体积 |

### 打包工具：electron-builder

- 自动处理 native 模块（better-sqlite3、sharp）的编译
- 生成 NSIS 安装程序（Windows）
- 支持自动更新（后续可选）

## 架构设计

### 进程模型

```
┌─────────────────────────────────────────┐
│              主进程 (Main)               │
│  ┌──────────┐  ┌──────────┐  ┌────────┐ │
│  │ 剪贴板监听 │  │ 数据存储  │  │ 自动清理│ │
│  │clipboard.js│  │ store.js │  │autoclean│ │
│  └──────────┘  └──────────┘  └────────┘ │
│  ┌──────────┐  ┌──────────┐  ┌────────┐ │
│  │ 系统托盘   │  │ 全局快捷键│  │开机自启 │ │
│  │  tray     │  │shortcut  │  │autostart│ │
│  └──────────┘  └──────────┘  └────────┘ │
│                                         │
│  ◄────────── IPC 通信 ──────────►       │
│                                         │
│           渲染进程 (Renderer)            │
│  ┌─────────────────────────────────────┐│
│  │         index.html + app.js         ││
│  │  ┌────────┐ ┌────────┐ ┌─────────┐ ││
│  │  │ 搜索栏  │ │ 卡片列表│ │ 设置面板 │ ││
│  │  └────────┘ └────────┘ └─────────┘ ││
│  └─────────────────────────────────────┘│
└─────────────────────────────────────────┘
```

全局快捷键唤醒时，主进程通过 `focused-input.js` 读取当前输入位置。应用启动时预热一个常驻 PowerShell helper，并主动预热一次输入区域探测路径，避免每次快捷键都重新启动 PowerShell 和加载 UIAutomation；helper 脚本写入临时 `.ps1` 后通过 `-File` 执行，避免长脚本触发 Windows 命令行长度限制。探测分为快慢两层：快速层只使用 Win32 `GetGUIThreadInfo` 读取活动文本光标 `hwndCaret/rcCaret`、原生焦点编辑控件 `hwndFocus` 矩形，以及终端窗口底部锚点；这三类不经过 UI Automation，是快捷键热路径。只有快速层没有结果时，才进入较短预算的 Windows UI Automation 兜底，读取焦点控件 `BoundingRectangle`。首次冷启动会等待 helper ready；常驻预热完成后，光标路径通常能在毫秒级返回。读取超时、失败或无输入区域时退回屏幕右下角。若快捷键从外部应用触发，即使面板当前可见，也会重新读取输入框并刷新粘贴目标；只有面板自身聚焦时再次按快捷键才隐藏面板。若成功识别输入目标，点击卡片时主进程写入剪贴板、隐藏面板、通过 helper 恢复目标窗口焦点并发送 `Ctrl+V` 完成自动粘贴；粘贴目标同时保留顶层窗口 `hwnd` 和原输入控件 `focusHwnd`，恢复时先激活顶层窗口，再尽量 `SetFocus` 回原输入控件，避免只恢复窗口但焦点没有回到输入框。恢复焦点时仅对最小化窗口执行 `SW_RESTORE`，避免把最大化窗口还原为普通窗口。发送粘贴键前必须确认目标窗口已重新成为前台窗口；若常规 Win32 激活被 Windows 前台锁拒绝，则使用 `WScript.Shell.AppActivate` 按目标进程兜底激活，仍失败时返回复制成功但不误报已粘贴。

### IPC 通道设计

| 通道 | 方向 | 用途 |
|------|------|------|
| `clips:getAll` | Renderer → Main → Renderer | 获取所有记录（分页、搜索） |
| `clips:pin` | Renderer → Main | 切换置顶状态 |
| `clips:delete` | Renderer → Main | 删除单条记录 |
| `clips:copy` | Renderer → Main | 复制某条记录到剪贴板 |
| `settings:get` | Renderer → Main → Renderer | 获取设置 |
| `settings:set` | Renderer → Main | 修改设置 |
| `clipboard:newItem` | Main → Renderer | 通知新记录到达 |

### 数据库 Schema

```sql
-- 剪贴板记录表
CREATE TABLE IF NOT EXISTS clips (
  id         INTEGER PRIMARY KEY AUTOINCREMENT,
  type       TEXT    NOT NULL,   -- 'text' | 'image'
  content    TEXT,               -- 文字内容（图片类型为空）
  image_path TEXT,               -- 原图路径（文字类型为空）
  thumb_path TEXT,               -- 缩略图路径
  created_at INTEGER NOT NULL,   -- 复制时间戳（毫秒）
  pinned     INTEGER DEFAULT 0   -- 0=普通 1=置顶
);

-- 设置表（键值对）
CREATE TABLE IF NOT EXISTS settings (
  key   TEXT PRIMARY KEY,
  value TEXT NOT NULL
);

-- 默认设置
-- retention_days: 5
-- shortcut: Ctrl+Shift+Z
-- autostart: false
```

### 数据流

```
用户复制内容
    │
    ▼
剪贴板轮询检测到变化 ──► 去重判断 ──► 存入数据库
    │                    (与上次相同则跳过)    │
    │                                         ▼
    │                              文本：存 content 字段
    │                              图片：存 image_path + thumb_path
    │                                         │
    │  通知渲染进程 ◄──────────────────────────┘
    │    │
    ▼    ▼
渲染进程更新卡片列表
```

### 存储路径

```
{userData}/                    ← electron app.getPath('userData')
├── clipboard.db              ← SQLite 数据库文件
└── images/
    ├── originals/            ← 原图
    │   └── {timestamp}.png
    └── thumbs/               ← 缩略图 (200×120)
        └── {timestamp}.png
```
